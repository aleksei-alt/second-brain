## Технические инсайты

**WAT-фреймворк:**
- Workflow (инструкции/SOP) → Agent (решения) → Tools (исполнение)
- Деплоишь W + T, не A. Бот у клиента — детерминированная автоматизация, не живой агент.
- "5 шагов × 90% точности = 59% успеха" → делать маленькие проверяемые шаги

**CLAUDE.md — обязателен первым:**
- Структура папок + роль + фреймворк перед любым проектом
- Без этого код в мусор быстро

**Plan Mode (/plan) перед нетривиальным билдом** — читаешь план, потом выполняет

**ultrathink** = 32k токенов thinking. Когда Claude дал плохой ответ 2 раза — добавить слово в промпт.

**Skills = переиспользуемые модули:**
- Написал "бот записи beauty" один раз → каждый следующий клиент = 20 мин кастомизации
- Nate запускает 4 агента параллельно каждое утро через skills

**MCP + .env:** API ключи только в .env, никогда в чате (попадают в историю разговора)

**Playwright CLI > Chrome DevTools MCP** для браузерной автоматизации (DevTools жрёт токены)

**Explicit allowlist в settings.json** вместо --dangerously-skip-permissions

**n8n → веб-приложение:** существующий workflow → Claude строит фронтенд → деплой на Vercel через GitHub. Webhook URL → env variable. Готовый шаблон продажи.

**RAG для локального бизнеса:** прайс + FAQ + фото → бот отвечает 24/7. Именно это и продаём.

**Webhook без авторизации = опасно:** без secret спамят, и ты платишь из своих API кредитов.

**N8N лицензия:** нельзя один инстанс для нескольких клиентов. Каждый клиент — свой n8n cloud аккаунт, ты входишь как разработчик. Или отдаёшь JSON + Loom.

## Продажи — ключевые инсайты

**Оффер — никогда не говорить "AI-бот":**
→ "Клиенты записываются к вам 24/7 пока вы спите. Ни одна заявка не теряется."

**LRP-фреймворк на discovery call:**
- Listen (слушаешь как описывают процесс)
- Repeat (повторяешь паттерн, подтверждаешь)
- Poke (квантифицируешь): "Чьи это часы? Сколько стоит час этого человека? Как часто это приводит к ошибкам?"

**Вопросы-триггеры:**
- "Если убрать одну еженедельную головную боль — какую?"
- "Где ошибки вызывают переделку или потерю клиента?"

**Математика прайсинга (показываешь на звонке):**
1. Сколько часов/нед на запись/ответы?
2. × стоимость часа × 4 = месячная стоимость проблемы
3. "Я убираю 60% этой нагрузки. Окупается за 5 месяцев."
Пример beauty: 5ч × €15 × 4 = €300/мес проблема → €100/мес решение = дёшево

**Trust comes before retainers.** Первые 3-5 клиентов = платная практика.
Закрывай не на €150 — закрывай на бесплатный пилот: "Сделаю прототип, посмотрим сколько запросов перехватит."

**Trojan horse:** найти агентства/консультантов которые уже работают с RU-бизнесом → 20% rev share за intro. Сделки через партнёра закрываются на 46% быстрее.

**PoC за 1.5 часа:** 15 мин схема → 1 час сборка → 15 мин Loom. Камера на себя — "люди нанимают людей, не безликие экраны."

**После первого клиента — кейс до/после:**
- До: сколько пропущенных запросов/нед?
- После 30 дней: сколько перехватил бот?
→ Это единственный sales tool для следующих клиентов.
